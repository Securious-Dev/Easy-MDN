2. JavaScript 심화 문법

2.1. 반복문

반복문은 여러 동작을 반복하는 경우 사용합니다. 반복문을 사용하게 되면 동일한 코드를 여러 번 반복할 수 있습니다.

반복문을 사용하는 방법에는 for문과 while문이 있습니다.

2.1.1. while문

    
      while (조건) {
        // 조건을 만족하는 경우 실행할 코드
      }

while문은 조건을 만족하는 경우 계속 실행됩니다.

조건이 참인 경우 계속 실행되기 때문에 무한 루프를 만들 수 있습니다.

    
      while (true) {
        // 무한히 실행할 코드
      }

위의 코드에서 while문의 조건이 항상 true 이므로 해당 while문은 무한히 실행됩니다.

대부분의 while문은 조건을 만족 시키는 경우를 while문 안에 삽입하여 사용합니다.

    
      let i = 1;
      while (i < 11) {
        console.log(i);
        i++;
      }

i의 경우 코드가 실행되며 계속 값이 변화하기 때문에 let 키워드로 선언해야 합니다.

위 코드의 결과는 1부터 10까지의 숫자가 콘솔로 출력됩니다. (콘솔창을 열어 결과를 확인해보세요)

i가 11이 되면 while문의 조건을 만족하지 않기 때문에 코드가 실행되지 않습니다.

2.1.2. for문

while문 보다 복잡한 형태를 갖지만 가장 많이 쓰이는 반복문 입니다.

    
      for (반복문 시작점; 반복문 조건; 각 반복 이후 수행될 것) {
        // 조건을 만족하는 경우 실행될 코드
      }

마찬가지로 1부터 10까지 콘솔창에 출력하도록 하는 for문은 다음과 같이 작성할 수 있습니다. (콘솔창을 열어 결과를 확인해보세요)

    
      for ( let i = 1; i < 11; i++ ) {
        console.log(i);
      }

i는 1부터 시작합니다. i가 1인경우 i < 11이라는 조건을 만족하기 때문에 for문 안에 있는 코드가 실행됩니다.

for문 안의 코드가 실행되고 난 뒤, i++을 수행하여 i는 2가 됩니다.

위의 과정을 반복하여 i가 11이 된다면 i < 11을 만족하지 않기 때문에 더이상 for문은 실행되지 않습니다.

결론적으로 반복문 시작점은 for문이 처음 실행되며 한 번만 실행되며 for문이 반복되는 동안 반복문의 조건과 각 반복 이후 수행될 것은 계속 실행됩니다.

for문에서 무한루프는 아래와 같이 만들 수 있습니다.

    
      for (;;) {
        // 무한히 실행할 코드
      }

하지만, 대부분의 무한 루프는 while을 이용해서 만들게 됩니다.

2.1.3. break/continue

break와 continue는 반복문을 중간에 중지 또는 건너뛸 수 있도록 해줍니다.

예를 들어, 1에서 10까지 짝수만 출력하고 싶은 경우에 사용할 수 있습니다.

    
      let i = 1;
      while (true) {
        if ( i > 10 ) {
          break; // i가 10을 초과한다면 반복문을 종료한다.
        }
        if ( i % 2 !== 0 ) {
          i++;
          continue; // 짝수가 아닌 경우 아래 코드(console.log())를 실행하지 않고 다음으로 넘어간다.
        }
        
        console.log(i);
        i++;
      }

위의 코드의 결과는 2, 4, 6, 8, 10입니다. (콘솔창을 열어 결과를 확인해보세요)

예시에서 봤듯이 break가 실행되면 반복문은 종료됩니다.

continue가 실행되면 반복문이 종료되는 것이 아니라 반복문 속에서 continue 뒤에 있는 코드들은 실행되지 않고 다시 반복문의 처음으로 돌아갑니다.

위의 예시에서 i가 1인 경우 조건에 해당하기 때문에 i++이 실행된 후 continue가 실행됩니다. continue가 실행되었기 때문에 continue 밑에 있는 console.log(i)와 i++은 실행되지 않고 다시 반복문의 처음(2번째 줄)으로 이동하게 됩니다.

2.2. 조건문

위의 예시에 if를 볼 수 있었습니다.

if는 조건문을 만들 때 사용합니다.

조건문은 조건에 따라 다른 행동을 취해야 할 때 사용합니다.

    
      if (조건) {
        // 조건이 참일 경우 실행할 코드
      }

if문과 else문을 이용해 A또는 B를 나타낼 수 있습니다.

    
      if (조건) {
        // 조건이 참일 경우 실행할 코드
      } else {
        // 조건이 거짓일 경우 실행할 코드
      }

만약, A또는 B또는 C를 나타내고 싶다면 if else를 사용할 수 있습니다.

    
      if (조건1) {
        // 조건1이 참일 경우 실행할 코드
      } else if (조건2) {
        // 조건2가 참일 경우 실행할 코드
      } else {
        // 조건1과 조건2가 모두 거짓일 경우 실행할 코드
      }

조건이 더 많아지더라도 else if를 사용하면 모두 작성할 수 있습니다.

하지만 가독성이 떨어지므로 많은 수의 조건을 처리해야 하는 경우에는 switch문을 참고하는 것을 추천드립니다.

(아래 코드에는 양의 정수 이외의 다른 값을 입력하는 경우에 대한 처리가 추가로 되어있습니다. 아래 코드는 if문이 어떤 상황에 쓰이며 어떻게 작성해야 하는지 이해하는 용도로 사용하세요.)

    
      const i = ;

      if (i % 2 === 0) {
        alert ('짝수(또는 0) 입니다'); // i가 0또는 짝수인 경우 실행
      } else if (i % 2 === 1) {
        alert ('홀수 입니다'); // i가 홀수인 경우 실행 (모든 홀수를 2로 나눈 나머지는 1이다)
      } else {

2.3. 함수

함수는 스크립트를 작성하다 보면 유사한 동작을 하는 코드가 여러 곳에서 필요할 때 주로 사용합니다.

함수는 function 키워드를 이용해 선언할 수 있습니다.

    
      function 함수이름(매개변수(Parameter)) {
        // 함수가 실행할 코드
        return 함수가 반환할 값;
      }

return을 이용해 해당 함수가 반환할 값을 정할 수 있습니다.

함수는 함수 내부에서 여러 연산을 통해 함수의 목적에 맞는 값을 생성하고 return하게 됩니다.

위의 형식에서 볼 수 있듯 함수는 인수를 받을 수 있으며 받아온 인수를 함수 내부에서 사용할 수 있습니다.

    
      function addNum(a, b) {
        const answer = a + b;

        return answer;
        // 또는 return a + b;
      }

addNum 함수는 a와 b라는 매개변수를 가지며 함수 내부에서 a + b를 수행하여 값을 구하고 return을 통해 반환합니다.

매개변수는 갯수에 상관없이 가질 수 있으며 매개변수를 갖지 않는 함수도 생성할 수 있습니다.

    
      function returnNothing() {
        alert('return nothing!');
      }

함수를 호출하는 방법은 함수의 이름과 인수를 작성하는 것입니다.

  • 매개변수(Parameter): 함수 선언 방식 괄호 사이에 있는 변수(함수 선언 시 쓰이는 용어)
  • 인수{Argument}: 함수를 호출할 때 매개변수에 전달되는 값(함수 호출 시 쓰이는 용어)

매개변수가 존재하지 않는다면 인수를 작성하지 않아도 되며, 매개변수를 1개 이상 필요로 하는 함수의 경우 모든 인수를 순서대로 작성해야 합니다.

매개변수가 많아지는 경우 함수의 기능을 세분화해 여러개의 함수를 만들더라도 매개변수의 갯수를 최소한으로 유지하는 것이 이상적인 함수입니다.

이상적인 함수는 한 가지 일만 해야하며 가급적 작게 만들어야 하기 때문입니다.

    
      function addNum(a, b) { // a, b는 매개변수
        const answer = a + b;

        return answer;
      }
      
      const result = addNum(1, 2); // 1, 2 함수에 전달하는 인수이며 addNum의 매개변수 a와 b에 할당된 후 연산을 수행하여 반환
      console.log(result); // 결과는 3
